# Developers Academy GitHub Guide

This guide supports an **issue-first, document-friendly, and progressively Git-enabled** workflow.

<div class="grid cards" markdown>

-   **Track project work**

    Find a deliverable, understand its acceptance criteria, claim it, and record progress.

    [Open the project-work guide](issues-and-deliverables.md)

-   **Make a repository contribution**

    Use GitHub Desktop to create a branch, inspect changes, commit, push, and open a pull request.

    [Open the first-PR guide](first-github-desktop-pr.md)

-   **Troubleshoot with Copilot**

    Start from a structured issue, generate hypotheses, test them, and record evidence.

    [Open the Copilot guide](copilot-troubleshooting.md)

</div>

## Four objects to remember

| Object | Purpose |
|---|---|
| Issue | A task, problem, decision, or deliverable that needs attention |
| Project | The shared view of what is ready, underway, blocked, or completed |
| Branch | A safe workspace for changing repository files |
| Pull request | A proposal to add reviewed changes to the shared project |
