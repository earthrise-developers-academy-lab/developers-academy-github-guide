# Start here

## Your first project session

1. Open your team's GitHub Project.
2. Find the issue assigned to you or select an available deliverable.
3. Read the objective, required artifact, and acceptance criteria.
4. Assign the issue to yourself when appropriate.
5. Move it to **In progress**.
6. Add a comment describing your first action.
7. Link any existing Google Doc, spreadsheet, notebook, or other working material.
8. Record blockers in the issue as soon as they affect progress.

!!! important
    Active work should have an issue. The working artifact may remain in Google Docs, Excel, a notebook, or another suitable tool while it is being developed.

## Ask a repository steward when

- You may be handling sensitive or restricted information.
- GitHub Desktop shows changes you did not make.
- You edited the wrong branch.
- A pull request contains unrelated files.
- A merge conflict appears.
- Copilot proposes a large or poorly explained change.
