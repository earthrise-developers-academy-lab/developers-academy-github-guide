# First GitHub Desktop pull request

Complete your first pull request in this guide repository or another designated practice repository.

## Workflow

1. Open the assigned issue and read the acceptance criteria.
2. Create an issue-linked branch such as `issue-12-add-participant-page`.
3. Open the repository in VS Code.
4. Create `docs/participants/<github-username>.md`.
5. Return to GitHub Desktop and inspect the diff.
6. Confirm only the intended file changed.
7. Commit with a specific message.
8. Publish or push the branch.
9. Preview the pull request.
10. Confirm the base branch is `main`.
11. Include `Fixes #<issue-number>` when merging should close the issue.
12. Request review.

Example participant page:

```markdown
# Participant name

## Project

Project name

## Role or area of interest

Brief description.

## GitHub onboarding

- [x] Located the assigned issue
- [x] Created a branch
- [x] Added this file
- [ ] Opened a pull request
```

Stop and ask for help if you edited `main`, the branch contains unrelated changes, GitHub Desktop reports a conflict, or you do not understand the displayed diff.
